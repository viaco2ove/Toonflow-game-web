# web端和安卓端的差异
按住语音输入这个适用安卓，但是不适用web.

### 一、核心交互逻辑重构（彻底适配鼠标操作）
| 设计环节 | 移动端（原设计） | Web端（优化后）         |
| :--- | :--- |:------------------|
| **触发方式** | 长按麦克风按钮 | 点击麦克风按钮（单次点击）     |
| **录音状态** | 按住=录音，松开=发送 | 点击=开始录音，再次点击=结束录音 |
| **取消方式** | 上滑取消 | 提供「取消」按钮，或点击空白处取消 |
| **发送逻辑** | 松开自动发送 | 识别完成后，自动「发送」      |

## 安卓
按住进行录音。松开发送。上滑取消

## web
压根没长按的功能！！！
Web 端鼠标没有「长按 + 滑动」的操作习惯，必须用「点击触发、点击结束」的无状态操作，降低学习成本。

游玩vue的业务逻辑太混乱了，直接删了。然后用调试的那个vue 标记一下是游玩模式还是调试模式就行了。安卓也是。
